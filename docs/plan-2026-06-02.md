# Plan: Trust-Release Milestone + Vocabulary Extensions

**Date:** 2026-06-02
**Author:** Mark + Claude
**Supersedes:** `prd-status-2-24-2026.md`, `plan-2-24-2026.md`

## Context

Since the Feb 24 status doc, the content-management ring of the PRD has largely
landed. The app is in solid demo shape, persistence is real (SwiftData local
SQLite, CloudKit opt-in), and the content-authoring story holds together
end-to-end. This doc captures the next milestone — what we're calling the
**trust release** — so we can look back on the inflection point.

### What shipped since Feb 24
- Persistent SwiftData storage (was `isStoredInMemoryOnly: true`)
- CloudKit sync wiring (opt-in, off by default)
- Scene editor (`SceneEditorView`) + Page editor (`PageEditorView`)
- AI generators: `TileSuggestionService`, `SceneGeneratorService`, `PageGeneratorService`
- JSON import/export of scenes (`SceneExporter`, `SceneImporter`, `SceneImportSheet`,
  `SceneTransferModels`) — round-trips `userImageData` for referenced tiles
- TileScript record + playback (`RecordedScript`, recording/playback overlays)
- Activity Log (basic usage visualization, MetricEvent-backed)
- Promoted tile *candidates* surfaced in Admin (not yet promotable to real tiles)
- Responsive grid + unified tray across iPhone/iPad
- Core-First scene + scenes-as-data + drop of PageModel/PageTileModel
- Age-grade in `SentencePromptBuilder` (`ageGradeLevel`)
- Provider abstraction + Apple FoundationModels evaluation (parked — guardrails)
- Custom Playful-3D tile artwork in progress (replaces ARASAAC for license clearance)

## What's still open (PRD-aligned)

Grouped by what each ring unblocks. This is the canonical "remaining work"
list as of today.

### Ring A — Security & trust (gates real-user testing)
- **API key in Keychain.** Currently `@AppStorage` / UserDefaults (`AppSettings.swift`).
- **Face ID / passcode gate on Admin tab.** No `LAContext` anywhere today.
- **Onboarding wizard.** Non-technical parent API-key flow; gates public hand-off.

### Ring B — Per-child personalization
- **Child profile** (name, age/grade, voice, max tiles). `ageGradeLevel` and a
  voice ID exist in scattered settings; no `ChildProfile` model ties them.
- **Custom tile images.** `TileModel.userImageData` + exporter round-trip exist,
  but no PhotosPicker/camera UI to set one.
- **Vocabulary extensions** (NEW — see design notes below). Therapist-created
  tiles in the global vocabulary, sharable across devices.

### Ring C — Therapist workflows
- **Session mode.** Temporary scene takeover with auto-revert.
- **Promoted tiles** (full promotion, not just candidate surfacing).
- **iCloud sync UX.** Code path exists but defaults off; no first-run picker.

### Ring D — Polish / pre-App-Store
- **Repetition prosody escalation** in TTS. Language escalates today;
  rate/pitch/volume don't (`SpeechSynthesizer` only sets the string).
- **Custom Playful-3D tile set replacement** for license clearance (in flight).
- **Localization.**

## Recommended order + justification

The shape of the next ring depends on whether we're heading toward
**first real-user testing** (Kurt's UX studies, family, friendly clinicians)
or **public release**. Real-user testing is the more urgent target — we
can't responsibly hand an iPad to a child/parent pair until Admin is gated
and the API key isn't sitting in plist-readable storage.

The order below treats steps 1–5 as the **trust release** milestone — a
coherent unit we'd ship together before opening up to outside testers.

| # | Item | Why this order |
|---|---|---|
| 1 | **Child profile model** | Small SwiftData entity (name, age/grade, voice, max tiles), wired into `SentencePromptBuilder.ageGradeLevel` and `SpeechSynthesizer`. Unblocks (2)'s "whose iPad is this" framing and (5)'s default-to-revert. |
| 2 | **Face ID / passcode gate on Admin** | Biggest trust gap. Cheap to build (`LAContext` + fallback PIN), big posture win. Required before handing the device to anyone. |
| 3 | **Keychain for API key** | Pairs naturally with (2); same security pass. |
| 4 | **Onboarding wizard** | First-run: profile → API key (Keychain) → optional CloudKit toggle → done. After this we can give a build to a non-Mark user without coaching. |
| 5 | **Vocabulary extensions + custom tile images** | Single sheet: add new tile (key, value, wordClass, image source = PhotosPicker / camera). Extends `SceneExporter` with a vocab-only payload variant + import-time conflict resolution + image downscale on ingest. Pair these because the UI primitive is identical. |
| 6 | **Session mode** | Therapist UX; needs (1) to feel right (per-child default scene to revert to). Smaller scope than the editors already shipped. |
| 7 | **Promoted tiles (real promotion)** | Needs accumulated usage. Better after 1–5 have been in real hands for a while. Current Admin candidate list is enough to validate the signal. |
| 8 | **Repetition prosody** | Small differentiator; do it once real users confirm the language escalation lands. |
| 9 | **Playful-3D art replacement + Localization + iCloud first-run UX** | App-Store gating ring. |

### Why this order, not "ship features in PRD-priority order"

- **Trust before reach.** Steps 1–5 are *prerequisites for putting the app
  in someone else's hands*, not features that make the app better in
  Mark's hands. Without them, every external test session requires Mark
  in the room.
- **Promoted tiles deferred deliberately.** It's a flagship feature, but
  the signal is only meaningful with accumulated real-user data. Building
  it before we have real users means building it against synthetic data.
- **Prosody deferred deliberately.** Language escalation already proves
  the repetition-as-volume-knob hypothesis. Prosody is amplification, not
  validation.

## Vocabulary extensions — design notes

This is the largest new item in the milestone, so capturing the design
calls up front.

### Goal
A therapist (or parent) can add a new tile to the device — e.g., `pinkfong`,
with a photo or AI-generated image — without code. Once added, the tile
lives in the **global vocabulary** and can be placed on any page in any
scene. Sharing a scene that references a custom tile carries the tile
(image + metadata) to the recipient device.

### Entry points (UI)
Both, sharing one sheet:
- **Inline in `TilePickerView`** — "can't find it? + Create new tile". Path
  of least friction for therapists in the moment.
- **Top-level Vocabulary section in Admin** — scene-independent place to
  curate the device's custom vocabulary.

### Key uniqueness + collisions
- `TileModel.key` is unique *by convention* (BootstrapLoader dedupes; no
  `@Attribute(.unique)` because CloudKit is incompatible — see memory).
  User-added tiles must follow the same rule.
- Normalize input: `"Pink Fong"` → `pinkfong` (lowercase, strip whitespace,
  ASCII-fold).
- On collision: offer to reuse the existing tile or pick a suffix
  (`pinkfong_1`). The `_1`/`_2` convention from the original Blaster is
  already PRD-blessed as "a hack but maybe useful if applied uniformly."

### Image source
v1: PhotosPicker + camera. AI-generated images via OpenAI is a deliberate
follow-up — the plumbing exists but adds API-call cost and latency to
the add-tile flow.

### wordClass for new tiles
Picker over existing classes (so color coding + prompt disambiguation
still work). Allowing creation of new classes is deferred — new classes
without code changes get no color and the prompt has no semantics for
them.

### Sharing
- **Scene-with-custom-tiles export already works** as a side effect.
  `SceneExporter` base64-encodes `userImageData` for any tile referenced
  by the exported scene.
- **New: vocabulary-only export.** Sibling payload type alongside
  `SceneTransferModels` — "share these 12 new tiles, no scene."
- **Conflict resolution on import.** If incoming `pinkfong` differs from
  the recipient's existing `pinkfong`, default to *prompt per-tile* with
  "do this for all" toggle. Options: prefer-incoming, prefer-existing,
  auto-suffix the incoming tile.
- **Image size cap.** Nothing constrains `userImageData` today. Wild
  imports could balloon. Downscale + JPEG-encode on ingest (target:
  512px max edge, JPEG quality ~0.85).

### Out of scope for v1
- AI-generated tile images
- Cross-class tile creation (creating new wordClass values)
- Bulk vocabulary import from spreadsheets / external symbol libraries
- Search across multi-device vocabulary libraries

## Definition of done for the trust release

Ship 1–5 together as a single milestone. Done means:
- Fresh-install on a new iPad: first-run wizard collects child profile,
  API key (stored in Keychain), and a CloudKit choice; lands the user on
  the Core-First home grid.
- Admin tab requires Face ID (or fallback PIN) on every cold launch and
  after backgrounding past a configurable timeout.
- A therapist can add `pinkfong` with a photo from the picker, drop it
  onto a page, AirDrop the scene to another device, and have the
  recipient see `pinkfong` (image and all) without re-adding the tile.
- An external tester can pick up the device with no Mark-in-the-room
  coaching required.

Once 1–5 are in real users' hands, revisit 6–9 with whatever signal we
get back.
