```markdown
## Current Architecture

### Data model (SwiftData, all in-memory for now)
All models live in `claudeBlast/Models/`.

- **TileModel** тАФ the vocabulary unit. `key` is unique ID and asset name (e.g. `eat`).
  `bundleImage == key` тЖТ `Assets.xcassets/{key}.imageset/{key}.png`.
  `value` is the display/speech string. `wordClass` is semantic category only (not layout).
  `userImageData: Data?` for future custom photos.

- **PageTileModel** тАФ junction of TileModel + page-specific behavior.
  `link: String` тАФ if non-empty, tapping navigates to this page key.
  `isAudible: Bool` тАФ if true, tap adds tile to sentence tray.

- **PageModel** тАФ ordered collection of PageTileModel.
  `tileOrder: [String]` (array of PageTileModel IDs) defines display order.
  `orderedTiles` computed property walks tileOrder to return ordered tiles.
  `displayName` doubles as the page's lookup key (e.g. `"home"`, `"actions"`).
  Methods: `removeTile(_:)`, `moveTile(from:to:)`.

- **BlasterScene** тАФ a named set of pages with a designated home page.
  `homePageKey: String` тАФ displayName of the page shown first when scene is active.
  `isDefault: Bool` тАФ the built-in vocabulary scene, never deleted.
  `isActive: Bool` тАФ only one scene active at a time; `activate(context:)` enforces this.
  `pages: [PageModel]` тАФ relationship (nullify delete rule).

- **SentenceCache** тАФ cached AI responses keyed by sorted tile combination.
  `cacheKey` = tile keys sorted alphabetically + joined (order-independent).
  `hitCount` tracks frequency (future: promoted tiles).

- **MetricEvent** тАФ raw event log for future analytics. Not yet surfaced in UI.

**Storage:** `isStoredInMemoryOnly: true` in `claudeBlastApp.swift:25` тАФ data wiped on
every launch. Intentional for now; switching to persistent + CloudKit is a planned chunk.

### Bootstrap flow
`BootstrapLoader.loadDefaultVocabulary(context:)` runs at app launch:
1. Decodes `Resources/vocabulary.json` тЖТ `[TileModel]` (480 tiles)
2. Decodes `Resources/pages.json` тЖТ `[PageModel]` with `PageTileModel` junctions
3. Creates "Default" scene (isDefault, isActive) with all pages
4. Creates "All Tiles (Review)" scene (flat single page, sorted by wordClass)
5. Inserts everything in one `context.transaction {}`

`vocabulary.json` format: `[{ "key": "eat", "wordClass": "actions" }]`
`pages.json` format: `[{ "key": "home", "pageTiles": [{ "key": "eat", "link": "food", "isAudible": false }] }]`
All tile keys must be lowercase. Keys map directly to imageset directory names.

### Tile images
`Assets.xcassets/{key}.imageset/{key}.png` тАФ one imageset per tile.
Source: ARASAAC (CC BY-NC-SA 4.0) for most tiles; DALL-E 3 for ~20 tiles with no good
ARASAAC match. Before App Store: replace all with custom DALL-E set to clear license.
Tools: `tools/download_arasaac.py`, `tools/generate_dalle.py`, `tools/prompts.json`.

### SentenceEngine (Observable, MainActor)
Lives in `Engine/SentenceEngine.swift`. Injected as environment object.
- `selectedTiles: [TileSelection]` тАФ current tray selection (max 4)
- Debounce: 350ms after last tile tap тЖТ triggers generation
- Cache-first: `SentenceCacheManager.lookup(tiles:)` тЖТ instant hit or API call
- Staleness guard: if tiles changed while API was in flight, discard result
- Conversational context: last 5 generated sentences fed back to AI as history
- Repetition escalation: same tile combination tapped repeatedly тЖТ escalating urgency in prompt
- Idle timeout: 30s after generation тЖТ `clearSelection()`
- `sessionNotes: String` тАФ free-text notes, long-press any tile to append

Provider abstraction: `SentenceProvider` protocol. Implementations:
- `OpenAISentenceProvider` тАФ gpt-4o-audio-preview, returns text + base64 audio
- `MockSentenceProvider` тАФ instant fake response, no API key needed

API key stored in `@AppStorage("openai_api_key")` (UserDefaults). Dev-only acceptable.
Override: set `OPENAI_API_KEY` env var in scheme тЖТ takes precedence, skips UI entry.

### Views
- `ContentView` тАФ root TabView with two tabs: child grid (TileGridView) and admin (AdminView)
- `TileGridView` тАФ paginated grid. `tilesPerPage(geo:isLandscape:)` computes from geometry
  (min tile 72pt). Portrait uses `VStack` (not LazyVStack) for correct snap offsets.
  Debug builds show breadcrumb navigation trail.
- `TileView` тАФ single tile: image + label. Color-coded by wordClass (see `wordClassColor()`
  in `SentenceTrayView.swift`). Long-press тЖТ note capture sheet.
- `SentenceTrayView` тАФ top tray: mini color-tinted tile chips + generated sentence text +
  replay/clear buttons. History sheet accessible via clock icon.
- `AdminView` тАФ admin panel: scenes list (activate/delete), cache viewer,
  session notes, provider/key picker, "Create Sample Scene" button.

### wordClass color mapping (file-private in SentenceTrayView.swift)
actions=orange, describe=green, people=purple, food/meals/fruit/veggie/snacks=red,
places=blue, social/feeling/question=pink, navigation=indigo, drinks=cyan,
weather=blue-gray, colors=mint, shape=teal, body/health=salmon, toy/games/sports=yellow,
art=purple-light, play=yellow, default=gray

### Pages structure (pages.json)
12 pages after merging continuation pages:
home(35), people(20), social(36), actions(108), describe(114), food(48),
drinks(11), places(36), play_activities(27), body_health(16), colors_shapes(24), weather(16)
No next_page/previous_page tiles тАФ app's built-in grid paging handles overflow.

### Branches
- `main` тАФ stable
- `demo-polish` тАФ current working branch (tile artwork, grid improvements)
- Feature work: always branch off main

### Known deferred items
- `isStoredInMemoryOnly: true` тАФ intentional, persistence is next planned chunk
- API key in UserDefaults тАФ Keychain deferred to pre-release
- Face ID gate on AdminView тАФ deferred to pre-release
- Scene/page editor тАФ next major feature chunk
- Child profile (name/age/voice) тАФ follows scene editor
```

### Chunk 1 тАФ Scene & Page Editor (ЁЯЯб, ~1 day)

The game-changing feature. Therapist creates a custom scene from the vocabulary.

**What exists:**
- `BlasterScene` model: name, descriptionText, homePageKey, pages, isDefault, isActive, activate()
- `PageModel`: displayName, orderedTiles, removeTile(), moveTile()
- `PageTileModel`: tile ref, link, isAudible
- `AdminView`: scene list with activate/delete, `createSampleScene()` as reference impl
- Full ~480 tile vocabulary in SwiftData after bootstrap

**Tasks:**

1. **SceneEditorView** (`Views/Editors/SceneEditorView.swift`)
   - Form fields: name (TextField), description (TextField), home page (Picker over scene's pages)
   - Page list with reorder + delete; "Add Page" button creates a new empty PageModel
   - Tap a page row тЖТ NavigationLink to PageEditorView
   - Toolbar: Save (insert into context + dismiss) / Cancel

2. **PageEditorView** (`Views/Editors/PageEditorView.swift`)
   - Header: editable page display name
   - Current tiles: List with drag-to-reorder handle + swipe-to-delete
     - Calls `page.moveTile(from:to:)` and `page.removeTile(_:)` тАФ methods already exist
   - "Add Tiles" button тЖТ sheet presenting TilePickerView

3. **TilePickerView** (`Views/Editors/TilePickerView.swift`)
   - Segmented/Picker: filter by wordClass
   - Search field: filter by key/displayName
   - Grid of TileView (read-only). Multi-select. "Add N Tiles" confirm button.
   - Returns selected TileModels тЖТ caller creates PageTileModel for each

4. **Wire AdminView**
   - "Create Scene" button тЖТ NavigationLink to `SceneEditorView(scene: nil)` (new scene)
   - Scene row: Edit button тЖТ `SceneEditorView(scene: existingScene)`
   - Remove `createSampleScene()` placeholder once editor exists

**AI assist (modest, built-in):**
For the AI assist on scene creation: a "Suggest tiles for..." text field in SceneEditorView.
Therapist types a goal ("emotions for a 6-year-old"). Calls OpenAI (text-only, not audio)
with a prompt to return a JSON array of tile keys from the vocabulary. App filters the
returned keys against the actual vocabulary, auto-selects matching tiles in TilePickerView.
This is optional stretch goal for this chunk тАФ do the manual editor first.
