Verify what runtime enforcement actually exists (system-prompt persona + content rails) vs. what's only eval-checked; separate the two claims. **The prompt rails ARE runtime; the *scoring* is not** — say each accurately.
- **Deliverable:** an audit table (claim → location → substantiated? → revised wording), then apply edits to deck + site (mirror the phrasing on the site's demo/about copy). Same rigor as the commissioned privacy review.

### A3 — Realistic cache policy + OpenAI terms review
Current `SentenceCache`: key = tile keys sorted + joined (order-independent), `+childID`, `hitCount`;
**no model version, no prompt version, no TTL/expiry**. Now CloudKit-synced, so cached OpenAI
outputs replicate across the family's devices via their iCloud.
- **Cache correctness:**
  - **Version the key** with model id + prompt/rubric version → a model or prompt change invalidates stale outputs instead of serving them forever. (Today a prompt fix like the escalation one would keep re-serving pre-fix cached sentences.)
  - **TTL / eviction** — decide a max age and a size/count bound with LRU (or hitCount-weighted) eviction; unbounded cache grows in CloudKit indefinitely.
  - **Escalation interaction** — repetition escalation is stateful; confirm the cache doesn't flatten escalated variants (they must key differently by repeat count).
- **OpenAI terms:**
  - Confirm storing + syncing generated outputs through the *user's own* iCloud is within OpenAI usage policies (generally: user owns outputs; verify no re-distribution concern when a cached output is served to a *different device/child*).
  - Reconcile with the public **"no external backend / stateless API"** claim — caching is local/iCloud, not our backend, so the claim holds, but state the nuance so A2 and A3 stay consistent.
  - Check data-retention / training-use posture of the API path we use and whether any opt-out flags should be set on requests.
- **Deliverable:** versioned cache key + TTL/eviction policy (design note + impl), and a short ToS-compliance memo. Touches `Models/SentenceCache.swift`, the cache manager, and the provider request path.
