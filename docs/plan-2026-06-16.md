### Gaps

**AI-generation quality (NEW — Mark's top concern):**
- **Escalation quality is poor today.** Text escalation fires, but the output
  isn't good enough. The real fix is upstream: we have no way to *measure* AI
  quality. This calls for an **AI-quality release + eval/test harness** that can
  run, score, and regression-test sentence generation, escalation, and scene
  generation. This is the gating quality work — prosody only makes sense *after*
  escalation quality is good (see Milestone 1.A and the prosody item).

**Partial / deliberately deferred:**
- **Promoted tiles** — only *surfaced* as candidates (cache hits ≥3); no actual
  promotion to real `TileModel` tiles. Deferred until real usage data exists.
- **Repetition prosody** — sentence *text* escalates, but `SpeechSynthesizer`
  rate/pitch/volume are static per child. Prosody doesn't escalate. Gated behind
  escalation-quality work.
- **Session mode** — not present, and likely *obsolete framing*. In the current
  app, "takeover" is arguably just: PIN into Admin → swap/import a scene → done.
  The PRD's auto-reverting session takeover predates child profiles + scene
  swapping. Action: validate whether anything beyond PIN→Admin→scene-swap is
  actually needed before building it (see Milestone 2).

**Missing / launch-gating:**
- **Playful-3D becomes the master tile set.** Only ARASAAC (496 imagesets) ships
  in-app today. Playful-3D (492 masters) and High Contrast (474) exist as LFS
  masters in `tools/tile_sets/` but aren't in `Assets.xcassets`. The
  `TileImageResolver` has a 3-set fallback chain, but it currently leans on
  ARASAAC as the backfill. Mark's call: **Playful-3D is the master** — the most
  complete vocabulary coverage and the highest tile quality — and it is the
  **backfill source** the resolver falls back to whenever a sparser swapped-in
  set (High Contrast, ARASAAC, a future set) is missing a tile. ARASAAC stays
  only as *one swappable option among others*, not the backfill. This requires
  bringing P3D to full vocabulary coverage and clears the App Store licensing
  gate (ARASAAC is CC BY-NC-SA). Moved into the pre-pilot sprint (Milestone 1.B).
- **Localization** — all UI hardcoded English. No String Catalog.
- **API-key onboarding polish** — key entry exists, but no visual setup guide /
  deep link / paste button / live validation / cost estimate. **Confirmed
  in-scope** for the pilot (Mark: "absolutely yes").
- **App Store prerequisites** — privacy policy, age rating, App Store metadata.

**Admin / logs UX (NEW — wanted before handoff):**
- `AdminView.swift` is 2,787 LOC. Beyond the maintenance risk, the **logs/usage
  functionality is weak and the widget for getting into Admin is awkward**. Mark
  wants this cleaned up *before* the TestFlight handoff, not deferred (moved into
  Milestone 1.D).

## Launch framing

The GTM doc defines three phases; the right "MVP" depends on the target:

- **Phase 1 — TestFlight pilot (5–10 SLPs).** GTM's framing was "polish and
  stability, not features," but the 6/17 review reframes the pre-pilot sprint as
  also covering AI quality, the art switch, and Admin cleanup. Default art for
  the pilot is **Playful-3D**, with **ARASAAC as a swappable option**.
- **Phase 3 — App Store (free).** Adds hard gates: privacy policy, age rating,
  parent-friendly key onboarding, localization. (Art licensing is already
  resolved by the Milestone 1 Playful-3D switch.)

Roadmap below hits **TestFlight MVP first**, then the App Store ring.

## Roadmap

### Milestone 1 — Pre-pilot sprint (TestFlight-ready)
*Goal: hand an iPad to a pilot SLP with no Mark-in-the-room. Now spans AI quality, the art switch, and Admin cleanup — not just cosmetic polish.*

**A. AI quality + eval harness (top priority)**
- Build testing/tooling to **run, score, and evaluate** sentence generation,
  escalation, and scene-generation AI interactions — a repeatable harness, not
  ad-hoc spot checks. Likely reuses TileScript's bulk-runner plumbing.
- Use it to **fix escalation quality** (the known weak spot) and lock in a
  regression baseline before the pilot.

**B. Make Playful-3D the master tile set**
- Import Playful-3D masters into `Assets.xcassets`; make it the **default** set.
- **Bring P3D to full vocabulary coverage** (currently 492 vs 496 keys) and best
  quality — it must be our most complete, highest-quality set.
- Re-point the `TileImageResolver` fallback so **P3D is the backfill** for any
  sparser swapped-in set (High Contrast, ARASAAC, future sets). ARASAAC drops to
  *one swappable option among others* — no special backfill role.
- *Investigate:* extending/refreshing the master set with a single uniform AI
  model + prompt for visual consistency — scope the **time and $$**. (Ballpark
  from the GTM doc: ~$20 in image-gen API for ~473 tiles using the existing
  `tools/generate_sets.py` pipeline, plus review/regen cycle time. To be
  confirmed.)

**C. Child-surface UX**
- **Make single-word usage first-class.** For a 1-tile selection, the Go/Play
  button becomes an **X with auto-clear** (speak the word, no API call), per the
  PRD's single-tile path.

**D. Admin / content cleanup (before handoff)**
- Refactor `AdminView.swift` into per-tab files; **improve the logs/usage**
  functionality; fix the **awkward Admin-entry widget**.
- Integrate TileScript into Admin; add **sample prompts in the scene builder**
  (and/or TileScript-driven scene generation) so therapists aren't staring at a
  blank prompt box.

**E. Onboarding & key (in-scope, confirmed)**
- **API-key onboarding polish** — deep link to platform.openai.com/api-keys,
  paste-from-clipboard, immediate key validation with a friendly error, one-line
  cost estimate. De-risks the GTM's #1 adoption barrier.

**F. Stability & ship**
- Pre-flight stability pass — clean `xcodebuild` + all 12 test targets green on
  iPad Pro M5 / iOS 26.2; fix any breakage.
- Cold-launch / fresh-install QA on a real device — full first-run wizard
  (Patient + Caregiver), Admin gate cold-launch, key entry, scene generate →
  child grid. The "external tester picks it up" DoD.
- **Demo video & deck** for pilot recruiting / SLP outreach.
- TestFlight build setup — archive scheme, App Store Connect record, privacy
  nutrition labels (trivial — no backend), invite cohort.

### Milestone 2 — Pilot-feedback ring (do *after* first sessions)
*These need real usage signal; building before the pilot means building against synthetic data.*

- **Repetition prosody escalation** — wire repetition count → rate/volume/pitch
  in `SpeechSynthesizer`. **Gated behind Milestone 1.A**: only worth it once the
  language escalation quality is actually good.
- **Promoted tiles (real promotion)** — materialize frequent combos into actual
  tiles. Best after pilot data accumulates; candidate surfacing already
  validates the signal.
- **Session mode — validate before building.** Confirm whether PIN→Admin→swap/
  import a scene already covers the "therapist takeover" need. Only build the
  auto-reverting session mode if a pilot SLP surfaces a concrete gap.

### Milestone 3 — App Store ring (Phase 3 gates)
- **App Store compliance** — privacy policy page, age rating, metadata /
  screenshots, accessibility-program application. (Art licensing already cleared
  by the Milestone 1.B Playful-3D switch.)
- **Localization** — String Catalog extraction; pick first non-English locale
  (ARASAAC has multilingual metadata to lean on).
- **Complete uniform custom art set** — if the Milestone 1.B investigation says
  it's worth it, generate the full ARASAAC-style uniform set here.

---

## Bottom line

The pre-pilot sprint (Milestone 1) is more substantial than "polish": it's an
**AI-quality + eval harness**, the **Playful-3D default-art switch**, an
**Admin/logs cleanup**, **single-word UX**, and **key-onboarding polish**, then
stability + TestFlight setup. Feature work that needs real usage signal (prosody,
real promoted tiles, and *validating* whether session mode is even needed) waits
for pilot feedback. App Store gates (compliance, localization, optional full
custom art set) are Phase 3.
